# sp-dto-gen

Công cụ .NET CLI tự động sinh C# DTO từ Stored Procedure trên SQL Server, sử dụng `sp_describe_first_result_set` — không cần thực thi SP thật.

## Cài đặt

```bash
dotnet tool install -g SpDtoGen
```

## Cách dùng

#### Sinh DTO cho một SP cụ thể
```bash
sp-dto-gen dto \
  --connection "Server=localhost;Database=MyDb;Trusted_Connection=true;" \
  --sp usp_Order_GetById \
  --output ./DTOs \
  --namespace MyApp.Application.DTOs
```

#### Sinh DTO cho nhiều SP theo pattern
```bash
sp-dto-gen dto --connection "..." --sp "usp_Order*" --output ./DTOs
```

#### Sinh DTO cho tất cả SP trong database
```bash
sp-dto-gen dto --connection "..." --output ./DTOs
```

#### Xem trước, không ghi file
```bash
sp-dto-gen dto --connection "..." --sp usp_Order_GetById --dry-run
```

#### Sinh `record` thay vì `class`
```bash
sp-dto-gen dto --connection "..." --sp usp_Order_GetById --record
```

## Các tùy chọn

| Option | Viết tắt | Mặc định | Mô tả |
|---|---|---|---|
| `--connection` | `-c` | *(bắt buộc)* | Connection string tới SQL Server |
| `--sp` | `-s` | *(tất cả SP)* | Tên SP hoặc wildcard (`usp_Order*`) |
| `--output` | `-o` | thư mục hiện tại | Thư mục xuất file `.cs` |
| `--namespace` | `-n` | `Application.DTOs` | C# namespace của DTO |
| `--suffix` | | `Dto` | Hậu tố thêm vào tên class |
| `--record` | | false | Sinh `record` thay vì `class` |
| `--dry-run` | | false | In ra console, không ghi file |
| `--force` | `-f` | false | Ghi đè file đã tồn tại |

## Ví dụ kết quả

Từ stored procedure:
```sql
CREATE PROCEDURE usp_Order_GetById @Id INT
AS
SELECT
    o.OrderId,
    o.CustomerName,
    o.TotalAmount,     -- decimal(18,2)
    o.CreatedAt,       -- datetime2
    o.IsDeleted,       -- bit
    o.AssignedUserId   -- int NULL
FROM Orders o
WHERE o.OrderId = @Id
```

Tool sinh ra file `OrderGetByIdDto.cs`:
```csharp
// <auto-generated>
//   Generated by SpDtoGen on 2024-01-15 08:30:00 UTC
//   Source SP: [dbo].[usp_Order_GetById]
// </auto-generated>

using System.CodeDom.Compiler;

namespace MyApp.Application.DTOs;

[GeneratedCode("SpDtoGen", "1.0.0")]
/// <summary>DTO for stored procedure [dbo].[usp_Order_GetById]</summary>
public class OrderGetByIdDto
{
    public int OrderId { get; set; }
    public string CustomerName { get; set; } = string.Empty;
    /// <remarks>SQL: decimal(18,2) — precision=18, scale=2</remarks>
    public decimal TotalAmount { get; set; }
    public DateTime CreatedAt { get; set; }
    public bool IsDeleted { get; set; }
    public int? AssignedUserId { get; set; }
}
```

## Quy tắc đặt tên

| Tên SP | Tên DTO được sinh |
|---|---|
| `usp_Order_GetById` | `OrderGetByIdDto` |
| `usp_GetAllProducts` | `GetAllProductsDto` |
| `sp_Customer_Insert` | `CustomerInsertDto` |
| `proc_Report_Monthly` | `ReportMonthlyDto` |

## Giới hạn đã biết

> ⚠️ **SP dùng temp table (`#tempTable`)** — `sp_describe_first_result_set` không thể phân tích
> tĩnh các SP có dùng temp table. Tool sẽ sinh ra skeleton rỗng kèm comment cảnh báo,
> bạn cần tự thêm property vào.
>
> **Cách khắc phục:** Cân nhắc viết lại SP để dùng table variable (`@tableVar`) thay cho
> temp table — loại này được hỗ trợ đầy đủ.

- **Dynamic SQL** (`EXEC(@sql)`) — tương tự, không thể phân tích tĩnh.
- **IF/ELSE trả về kết quả khác nhau** — chỉ lấy được nhánh đầu tiên.
- **Nhiều result set** — chỉ sinh DTO cho result set đầu tiên.
