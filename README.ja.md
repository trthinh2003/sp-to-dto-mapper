# sp-dto-gen

SQL ServerのストアドプロシージャからC# DTOを自動生成する .NET CLIツールです。`sp_describe_first_result_set` を使用して静的解析するため、**SPを実際に実行する必要はありません**。

## インストール

```bash
dotnet tool install -g SpDtoGen
```

## 使い方

#### 単一のSPからDTOを生成
```bash
sp-dto-gen dto \
  --connection "Server=localhost;Database=MyDb;Trusted_Connection=true;" \
  --sp usp_Order_GetById \
  --output ./DTOs \
  --namespace MyApp.Application.DTOs
```

#### ワイルドカードパターンで一致するSPのDTOを生成
```bash
sp-dto-gen dto --connection "..." --sp "usp_Order*" --output ./DTOs
```

#### データベース内の全SPのDTOを生成
```bash
sp-dto-gen dto --connection "..." --output ./DTOs
```

#### ファイルを書き込まずにプレビュー
```bash
sp-dto-gen dto --connection "..." --sp usp_Order_GetById --dry-run
```

#### `class` の代わりに `record` を生成
```bash
sp-dto-gen dto --connection "..." --sp usp_Order_GetById --record
```

## オプション一覧

| オプション | 短縮形 | デフォルト | 説明 |
|---|---|---|---|
| `--connection` | `-c` | *(必須)* | SQL Server 接続文字列 |
| `--sp` | `-s` | *(全SP)* | SP名またはワイルドカード (`usp_Order*`) |
| `--output` | `-o` | カレントディレクトリ | `.cs` ファイルの出力先ディレクトリ |
| `--namespace` | `-n` | `Application.DTOs` | 生成DTOのC# 名前空間 |
| `--suffix` | | `Dto` | クラス名に付加するサフィックス |
| `--record` | | false | `class` の代わりに `record` を生成 |
| `--dry-run` | | false | ファイルを書き込まずコンソールに出力 |
| `--force` | `-f` | false | 既存ファイルを上書き |

## 出力例

以下のストアドプロシージャがあるとします：
```sql
CREATE PROCEDURE usp_Order_GetById @Id INT
AS
SELECT
    o.OrderId,
    o.CustomerName,
    o.TotalAmount,     -- decimal(18,2)
    o.CreatedAt,       -- datetime2
    o.IsDeleted,       -- bit
    o.AssignedUserId   -- int NULL
FROM Orders o
WHERE o.OrderId = @Id
```

生成される `OrderGetByIdDto.cs`：
```csharp
// <auto-generated>
//   Generated by SpDtoGen on 2024-01-15 08:30:00 UTC
//   Source SP: [dbo].[usp_Order_GetById]
// </auto-generated>

using System.CodeDom.Compiler;

namespace MyApp.Application.DTOs;

[GeneratedCode("SpDtoGen", "1.0.0")]
/// <summary>DTO for stored procedure [dbo].[usp_Order_GetById]</summary>
public class OrderGetByIdDto
{
    public int OrderId { get; set; }
    public string CustomerName { get; set; } = string.Empty;
    /// <remarks>SQL: decimal(18,2) — precision=18, scale=2</remarks>
    public decimal TotalAmount { get; set; }
    public DateTime CreatedAt { get; set; }
    public bool IsDeleted { get; set; }
    public int? AssignedUserId { get; set; }
}
```

| `usp_Order_GetById` | `OrderGetByIdDto` |
| `usp_GetAllProducts` | `GetAllProductsDto` |
| `sp_Customer_Insert` | `CustomerInsertDto` |

## 既知の制限事項

> ⚠️ **一時テーブル (`#tempTable`) を使用するSP** — `sp_describe_first_result_set` は
> 一時テーブルを使用するSPを静的解析できません。空のスケルトンと警告コメントが生成されるため、
> プロパティは手動で追加する必要があります。
>
> **回避策:** 一時テーブルの代わりにテーブル変数 (`@tableVar`) を使用するようSPを
> 書き直すことを検討してください。テーブル変数は完全にサポートされています。

- **動的SQL** (`EXEC(@sql)`) — 同様に静的解析不可。
- **IF/ELSEによる分岐** — SQL Serverが解決できる最初のブランチのみ使用。
- **複数の結果セット** — 最初の結果セットのみ生成。
